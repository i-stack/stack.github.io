CocoaPods制作

Posted on 2017-01-18 | In Objective-C , Other , CocoaPods制作

创建自己的 github 仓库

  1. 登录 gitHub, start a project；
  2. create a new repository；
  • Repository name 仓库名称,必填
  • Description 仓库的描述信息,可选
  • 仓库的公开性 Public 免费, Private 收费
  • 是否创建一个默认的 README文件, 一个完整的仓库都需要
  • 是否添加.gitignore 文件, 文件记录了若干种文件类型
  • license 类型 必须 这里使用 MIT, 用于验证 pod 是否可用
  1. 填写完成后 点击 Create repository 创建仓库结束。

clone 仓库到本地

方便向仓库中删减内容,需要先将仓库 clone 到本地

  1. git clone https://github.com/"username"/"repository name”；
  2. GitHub Desktop。
    操作完成后, GitHub 上对应的文件都会拷贝到本地。

向本地 git 仓库添加创建 Pods 依赖库所需文件

后缀为.podspec 文件 pod 依赖库描述文件,每个 pod 都会有一个

如何创建 pod spec

pod spec create QCMobileAdLibs

提交文件到 github

  1. set the new version to 1.0.0
  2. set the new tag to 1.0.0
  3. pod lib lint 如果没有出现警告和错误则成功,成功后显示

->QCMobileAdLibs(1.0.0)

QCMobileAdLibs passed validation

  1. 本地 git 仓库修改内容上传到 GitHub 仓库
  • git add -A && git commit -m “Release 1.0.0”
  • git tag ‘1.0.0’

git push –tags 这里会让输入用户名(git用户名,不是登录名)和密码(登录密码)

git push origin master

上传 pod spec文件到 CocoaPods 官方仓库

trunk 方式需要在 pod 0.33及以上版本；
PS:更新 pod:sudo gem install cocoapods。

注册 trunk

pod trunk register ‘git 邮箱’ ‘用户名’；
pod trunk me 输出下面信息表示注册成功。

配置 PodSpec

  1. 添加 podspec 描述文件；
  2. 通过 trunk 推送 pod spec 文件。

cd 到podSpec所在目录。

执行完成后提交 git 团队审核, 一般2日。

更新本地 pod 依赖

在 trunk push 后, 先用”pod search”搜索下,若没有则执行”pod setup”进行本地依赖库更新；
cd到~/.cocoapods目录，用du -sh *来查看下载进度。

更新仓库

  1. 修改完成后上传 git
  2. set new version to 1.0.1
  3. set new tag to 1.0.1
  4. pod lib lint
    -> QCMobileAdLibs (1.0.1)
    QCMobileAdLibs passed validation.

出现上面的内容表示成功,此时 git 上的版本也已更新；

  1. 修改.podspec 中版本号；
  2. pod trunk push .podspec。

类是什么?

类应该具有以下特性:

  1. 类能够完整的做完一件事, 目的很明确, 就是为了完成某件事情而产生;
  2. 由实例变量, 属性, 方法, 协议中一个或多个构成;
  3. 类也是一个对象即类对象, 类对象是一种数据结构, 存储类的大小, 类名称, 类的版本, 继承层次, 以及消息与函数的映射表;
  4. 类对象只能调用类方法;
  5. 类描述对象的行为;

堆和栈

Posted on 2017-01-15 | In Objective-C , Other , 堆和栈

栈和堆存储在哪?

他们都存储在电脑的 RAM(Random Access Memory) 中

栈和堆中都保存的是什么?

  1. 栈 —>编译器在需要的时候分配，在不需要的时候自动清除变量的存储区，通常是局部变量，函数参数等；
  2. 堆 —>由 new 分配的操作块，一个 new 对应一个 delete
  3. 自由存储区 —>由 malloc 分配的内存块
  4. 全局/静态存储区 —>全局变量和静态变量被分配到同一块内存中
  5. 常量存储区 —>这时一块比较特殊的存储区，存放常量，不能修改

看到 new 说明分配了一块堆内存，那么指针 p 是什么? p 分配的是一块栈内存，这句意思是在栈内存中存放一个指向一块堆内容的指针 p。程序先会确定在堆中分配内存的大小，然后调用 operator new 分配内存，让后返回这块内存的首地址，放入栈中。

栈和堆区别是什么?

  1. 管理方式不同：对于栈来说，是由编译器自动管理，无需我们手动控制；对于堆来说，释放工作由程序员控制，容易产生 memory leak；
  2. 空间大小不同:栈空间大小是1M，而在32位系统下，堆内存空间可以达到4G；
  3. 产生碎片不同:对于堆来说，频繁的 new/delete 势必会造成内存空间的不连续，而从造成大量碎片，使程序效率降低；对栈来说，不会产生碎片问题，栈是先进后出的队列不可能有一个内存块从栈中间弹出，在他弹出之前，在他上面的栈内容已经弹出；
  4. 生长方式不同:对于堆来说，生长方向是向上的，也就是向着内存地址增加的方向；对于栈来说，它的生长方向是向下的，是向着内存地址减少的方向增长；
  5. 分配方式不同:堆是动态分配，没有静态分配的堆；栈有2中分配方式:静态分配和动态分配，静态分配是编译器完成的，比如局部变量的分配；动态分配由 alloc 函数进行分配，由编译器进行释放；
  6. 分配效率不同:栈是机器系统提供的数据结构，计算机会在底层对栈提供支持:分配专门的寄存器存放栈地址，压栈出栈都有专门的指令执行，这就决定栈的效率比较高；堆是 C/C++函数提供的，例如分配一块内存，库函数会按照一定的算法在堆内存中搜索可用的足够大小的空间，如果没有足够大小的空间(可能由于内存碎片太多)，就有可能调用系统功能去增加程序数据段的内存空间，这样就有机会分到足够大小的内存，然后进行返回。

A+B问题

Posted on 2017-01-01 | In Arithmetic , A+B问题

给出两个整数a和b, 求他们的和, 但不能使用 + 等数学运算符.

  • a和b都是整数
  • 可以使用位运算符

样例

如果 a = 1 并且 b = 2, 返回3

解题思路分析

根据题目要求可以知道这是一道求和问题, 很容易联想到数学运算符 +, 这样就比较简单, 题目中要求不能使用数学运算符, 只能想其他的方式来解决. 又知a,b是整数, 可以使用位运算符, 这就可以确定这道题考察的是位运算符操作.

温习知识点: 位运算符

什么是位运算符?

位操作是程序设计中对位模式按位或二进制数的一元和二元操作.

位运算符有哪些?

  • & 按位与
  • | 按位或
  • ^ 按位异或
  • ~ 取反
  • << 左移
  • >> 右移

按位与运算

  1. 按位与运算符 & 是双目运算符;
  2. 两个二进位均为1时, 结果位才为1, 否则为0;
  3. 参与运算的数以 补码 方式出现;
  4. 按位与运算通常用来对某些位清0或保留某些位. 例如把 a 的高八位清0, 保留低八位, 可作 a & 255 运算(255的二进制数为11111111).
    例如: 9 & 5可写算式如下: 00001001(9的二进制补码) & 00000101(5的二进制补码) 00000001(1的二进制补码) 可见 9 & 5 = 1.

按位或运算

  1. 按位或运算符 | 是双目运算符;
  2. 二进位有一个为1时，结果位就为1;
  3. 参与运算的两个数均以 补码 出现.

按位异或运算

  1. 按位异或运算符 ^ 是双目运算符;
  2. 当两对应的二进位相异时, 结果为1;
  3. 参与运算数仍以补码出现。

求反运算

  1. 求反运算符 ～ 为单目运算符具有右结合性;
  2. 对参与运算的数的各二进位按位求反.

左移运算

  1. 左移运算符 << 是双目运算符;
  2. 左移n位就是乘以2的n次方;
  3. 其功能把 << 左边的运算数的各二进位全部左移若干位, 高位丢弃, 低位补0;
  4. 有符号数不完全适用, 因为左移有可能导致符号变化, 造成溢出;
  5. 总之左移就是: 丢弃最高位, 0补最低位;
  6. 最高位 0 正 1 负.

右移运算

  1. 右移运算符 >> 是双目运算符;
  2. 右移n位就是除以2的n次方
  3. 其功能是把 >> 左边的运算数的各二进位全部右移若干位;
  4. 最高位 0 正 1 负.
  • 对于有符号数在右移时符号位将随同移动;
  • 当为正数时最高位补0;
  • 当为负数时符号位为1;
  • 最高位是补0或是补1取决于编译系统的规定;
  • 右移对符号位的处理和左移不同: 对于有符号整数来说, 比如int类型, 右移会保持符号位不变.

异或操作的妙用

  1. 使特定位翻转 要使哪几位翻转就将与其进行∧运算的该几位置为1即可;
  2. 与0相∧, 保留原值;

  3. 交换两个值, 不用临时变量. 我们可以在不用引入其他变量就可以实现变量值的交换用异或操作可以实现:

  4. 异或操作满足结合律和交换律, 且由异或操作的性质知道——对于任意一个整数 a^a = 0.

位与运算

  1. 清零 A数中为1的位, B中相应位为0. 然后使二者进行&运算, 即可达到对A清零目的;
  2. 取一个数中某些指定位取数A的某些位, 把数B的某些位置1, 就把数A的某些位与1按位与即可;
  3. 保留一位的方法数A与数B进行&运算, 数B在数A要保留的位1, 其余位为零;
  4. 判断奇偶性: a与1做位与运算, 若结果是1, 则a是奇数; a与1做位与运算, 若结果是0, 则a是偶数.
