CocoaPods制作

发表于 2017-01-18 | 分类于 Objective-C ， CocoaPods | 浏览 次

  • 创建github仓库
  • clone仓库到本地

创建github仓库

  1. 登录 gitHub, start a project；
  2. create a new repository；
  • Repository name 仓库名称,必填
  • Description 仓库的描述信息,可选
  • 仓库的公开性 Public 免费, Private 收费
  • 是否创建一个默认的 README文件, 一个完整的仓库都需要
  • 是否添加.gitignore 文件, 文件记录了若干种文件类型
  • license 类型 必须 这里使用 MIT, 用于验证 pod 是否可用
  1. 填写完成后 点击 Create repository 创建仓库结束。

clone仓库到本地

方便向仓库中删减内容,需要先将仓库 clone 到本地

  1. git clone https://github.com/"username"/"repository name”；
  2. GitHub Desktop。
    操作完成后, GitHub 上对应的文件都会拷贝到本地。

添加Pods依赖库所需文件

后缀为.podspec 文件 pod 依赖库描述文件,每个 pod 都会有一个

如何创建 pod spec

pod spec create QCMobileAdLibs

提交文件到 github

  1. set the new version to 1.0.0
  2. set the new tag to 1.0.0
  3. pod lib lint 如果没有出现警告和错误则成功,成功后显示

->QCMobileAdLibs(1.0.0)

QCMobileAdLibs passed validation

  1. 本地 git 仓库修改内容上传到 GitHub 仓库
  • git add -A && git commit -m “Release 1.0.0”
  • git tag ‘1.0.0’

git push –tags 这里会让输入用户名(git用户名,不是登录名)和密码(登录密码)

git push origin master

上传 pod spec文件到 CocoaPods 官方仓库

trunk 方式需要在 pod 0.33及以上版本；
PS:更新 pod:sudo gem install cocoapods。

注册 trunk

pod trunk register ‘git 邮箱’ ‘用户名’；
pod trunk me 输出下面信息表示注册成功。

配置 PodSpec

  1. 添加 podspec 描述文件；
  2. 通过 trunk 推送 pod spec 文件。

cd 到podSpec所在目录。

执行完成后提交 git 团队审核, 一般2日。

更新本地 pod 依赖

在 trunk push 后, 先用”pod search”搜索下,若没有则执行”pod setup”进行本地依赖库更新；
cd到~/.cocoapods目录，用du -sh *来查看下载进度。

更新仓库

  1. 修改完成后上传 git
  2. set new version to 1.0.1
  3. set new tag to 1.0.1
  4. pod lib lint
    -> QCMobileAdLibs (1.0.1)
    QCMobileAdLibs passed validation.

出现上面的内容表示成功,此时 git 上的版本也已更新；

  1. 修改.podspec 中版本号；
  2. pod trunk push .podspec。
