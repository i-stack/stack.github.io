load

Invoked whenever a class or category is added to the Objective-C runtime; implement this method to perform class-specific behavior upon loading.

  1. load方法定义在NSObject.h头文件中；
  2. load方法只要被声明无论是否用到都会被加载；
  3. load方法在main函数之前调用；
  4. 父类的load方法调用优先于子类；
  5. 类别的load方法调用在当前类之后。

call_load_methods

这个方法主要做了哪些事情？

  1. 循环调用类的 + load 方法，直到 loadable_classes 为空；
  2. 调用一次 call_category_loads 加载分类；
  3. 如果有 loadable_classes 或者更多的分类，继续调用 load 方法。

这里出现两个方法：call_class_loads和call_category_loads
call_class_loads获取类中的load方法；
call_category_loads获取类别中的load方法。

call_class_loads

  1. 获取当前可加载的类列表；
  2. 调用类列表中的load方法；
  3. 释放类列表。

call_category_loads

load 调用

  1. 如果在一个类中实现 load 方法, 则会先调用其父类的 load 方法, 是系统自动调用的;
  2. 如果子类没有实现 load 方法, 则系统不会调用它父类的load方法;
  3. 类别中调用 load 方法, 遵循上面两条;
  4. load 方法不能手动调用;
  5. load 是线程安全的, 内部使用了锁, 应该避免在 load 中阻塞线程;
  6. 当Objective-C运行时初始化会通过 dyld_register_image_state_change_handler 在每次有新的镜像加入运行时进行回调。执行load_images 将所有包含 load 方法的文件加入列表 loadable_classes，然后从这个列表中找到对应的 load 方法的实现，调用 load 方法。

调用顺序: load_images -> prepare_load_methods -> schedule_class_load -> add_class_to_loadable_list 的时候会将未加载的类添加到 loadable_classes 数组中：

headerType 是一个结构体类型，包含load的类型，数量，大小等信息。

prepare_load_methods

  1. 获取类列表，调用schedule_class_load将父类未加载的load添加到loadable_classes中。
  2. 获取分类列表，将分类添加到loadable_categories中。

schedule_class_load

  1. 通过schedule_class_load(cls->superclass) 总是能够保证没有调用 load 方法的父类先于子类加入 loadable_classes 数组。
  2. 这也是为什么父类的load方法优先于子类的原因。

loadable_classes

  1. 获取当前类的load方法，如果不存在，直接return。
  2. 重新给 loadable_classes 分配空间；
  3. 保存当前的load方法到 loadable_classes中。

loadable_category

  1. 获取当前类别的load方法，如果不存在，直接return；
  2. 获取当前类别的名称；
  3. 重新给 loadable_categories 分配空间；
  4. 保存当前类别的load方法到 loadable_categories中。

headerType

load 使用场景

在主函数运行之前，load 方法就会调用。由于它的调用不是惰性的，且其只会在程序调用期间调用一次，最最重要的是，如果在类与分类中都实现了 load 方法，它们都会被调用。在此之前项目依赖的所有 framework 都已经加载到了运行时中，所以调用 framework 中的方法都是安全的。

常见的使用场景：Method Swizzling
