@objc

让 OC 能够调用该类，需要在类的定义前加入一个关键字 @objc。这是告诉编译器在幕后进行一些额外操作（命名，创建一个不可见的头文件等），让 Swift 的代码暴露给 OC。
只有一个XXX-Swift.h 头文件，所有通过 @objc 关键字暴露的 Swift 代码都会被放置这个 XXX-Swift.h 文件中。

Generated Interface Header Name

target -> Build Setting 选项卡。在 Swift Compiler – Code Generation 里找到 Objective-C Generated Interface Header Name：”XXX-Swift.h”。这就是 OC 中需要引入的头文件。

XXX-Bridging-Header.h

是用来暴露 OC 的库给 Swift 的。
