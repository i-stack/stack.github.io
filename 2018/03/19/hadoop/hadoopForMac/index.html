Hadoop for mac

发表于 2018-03-19 | 分类于 Hadoop ， Hadoop for mac | 浏览 次

什么是Hadoop?

  1. 开源，可靠的，可扩展的分布式计算开发开源软件；
  2. 作者：Doug Cutting；
  3. 受Goole三篇论文启发（GFS，MapReduce，BigTabel）。

解释：

  • 开源：说明是免费；
  • 可靠： 数据由备份，可自动恢复
  • 可扩展：灵活性高

HDFS基本概念

  1. Java实现，分布式，可横向扩展的分布式文件系统
  2. HDFS是Hadoop核心
  3. 基于unix，linix系统
  4. 容错性高，高吞吐量的数据访问
  5. 简单一致性模型
    • 一次写入多次查询，写入文件不能修改，如需修改先删除再写入新数据
    • 不支持并发写入
    • 小文件不适合
  6. 硬件错误是常态
  7. 流式数据访问，适用于处理批量，不适合随机定位访问
  8. 存储超大文件， PB/EB，单个文件一般在百MB以上
  9. 采用就近原则分配节点执行
    • 有ABC三台服务器，在A中执行下载任务，则在A上分配节点，不会在其他服务器上分配
  10. 运行在廉价的服务器上
  11. HDFS局限性
    • 不适合低延迟数据访问
    • 高吞吐量可能要求以高延迟作代价
    • 无法高效存储小文件
    • 文件数量限制由NameNode决定
    • 不支持多用户写入与修改
  12. 冗余副本策略
    • hdfs-site.xml 设置指定副本数量，所有数据库都有副本
  13. 机架策略
    • 集群一般在不同的机架上
    • 机架间带宽 < 机架内带宽
  14. 心跳机制
    • NameNode周期性从DataNode接收心跳信号和块报告
    • NameNode根据块报告验证元数据
    • 没有按时发送心跳的DataNode会被标记为宕机，不再有I/O请求
  15. 块
    • HDFS基于存储是块，块是读写的最小数据单位
    • Hadoop 1.x 默认64M
    • Hadoop 2.x默认128M
    • 存储200M文件
      • 1.x 分四个块 3个64M块，一个8M块
      • 2.x 分两个块 1个128M块，一个72M块
    • 块大小可自行配置，建议使用默认
      • 每个块都有一个自己的全局ID
      • 每个块都是独立存储单位
    • hdfs-site.xml 配置
      • 1.x 对应 dfs.block.size
      • 2.x 对应 dfs.blocksize
    • 一份元数据，两份备份，可以动态修改备份数
  16. HDFS元数据
    • 文件系统目录树信息
      • 文件名，目录名
      • 文件目录从属关系
      • 文件目录的大小，创建及最后访问时间
      • 文件目录的权限

Hadoop核心

  1. HDFS：Hadoop Distributed File System 分布式文件系统
  2. YARN：Yet Another Resource Negotiator 资源管理调度系统
  3. Common：公共模块，为其他Hadoop模块提供基础设施
  4. MapReduce：处理数据，一个分布式并行框架
    • 读取输入文件内容，解析成key, value对
    • 重写map方法，编写业务逻辑输出新的key, value对
    • 对输出的key, value分区
    • 对数据按照key 排序，分组，相同key的value放到一个集合中

HDFS的架构

  1. 主从结构
    • 主节点，只有一个：NameNode
    • 从节点，有很多个：DataNode
  2. NameNode负责：
    • 接收用户操作请求
    • 维护文件系统的目录结构
    • 管理文件与block之间的关系，block与 DataNode 之间的关系
  3. DataNode负责：
    • 存储文件
    • 文件被分成block存储在磁盘上
    • 为保证数据安全，文件会有多个副本
    • 启动时遍历本地文件系统，产生一个hdfs数据块和本地文件关系列表
  4. SecondaryNameNode负责：辅助 NameNode
    • 所在物理节点存储
    • 一次写入多次读取
    • 文件由数据块组成 64M，128M
    • 数据块尽量分布各节点

特点：

  1. 扩容能力：
    • 能可靠的存储和处理千兆（PB）字节数据
    • 动态新增及卸载机器
  2. 成本低：
    • 普通机器组成的服务器集群
  3. 高效率：
    • 通过分发数据，Hadoop可以在数据所在的节点上并行处理
  4. 可靠性：
    • Hadoop能自动维护数据的多份副本，并且在任务失败后能自动的重新部署计算任务，恢复数据
  5. 并行读写数据
    • 一块硬盘的读写性能 < 多块硬盘的读写性能
    • 1HDD = 75 MB/sec
    • 1000HDD = 75GB/sec

解决问题及适用场景

解决问题

  1. 海量数据的存储（HDFS)
    • 可动态新增机器提升空间
    • 自动备份，默认备份3份，在可配置文件中修改备份数
  2. 海量数据的分析（MapReduce）
    • 分而治之，比如分析100G数据通过多台节点来并行分析处理
  3. 资源管理调度（YARN）
    • 集群之间内存调度

适用场景

  1. 擅长日志分析
  2. 基于海量数据的在线应用
  3. 推荐系统
  4. 计算广告
  5. 复杂算法
  6. 网盘
  7. 搜索引擎

Hadoop部署方式

  1. 本地模式
  2. 伪分布模式
    • 关闭防火墙
    • 修改IP
    • 修改hostname
    • 设置ssh自动登录
    • 安装jdk
    • 安装hadoop
  3. 集群模式
